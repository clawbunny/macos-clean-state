macOS Clean State

A simple, powerful tool for capturing, managing, and restoring clean macOS device states using native APFS snapshots and compressed DMG images.

Perfect for developers, QA engineers, and power users who need reproducible "clean" starting points before running tests or experiments that pollute the system (installing apps, changing settings, running malware analysis, etc.).

Why This Tool?

Modern macOS testing often requires resetting a device to a known-good state after each test run. Traditional approaches (full Time Machine restores, manual cleanup, or VM snapshots) are either slow, incomplete, or require heavy virtualization.

clean-state.sh leverages macOS's built-in APFS snapshot technology for near-instant captures and restores, combined with easy export to portable compressed disk images.

Key Benefit: A snapshot takes seconds and uses almost zero extra space until files change. You can create a clean baseline, run destructive tests, then revert in Recovery in minutes.

How It Works

1. Local APFS Snapshots (Fast & Lightweight)

  • Uses tmutil localsnapshot to create point-in-time copies of your Data volume.
  • Snapshots live on the same drive and are extremely efficient (copy-on-write).
  • Named in the standard Time Machine format so they appear in Recovery's "Restore from Time Machine Backup" interface.

2. Human-Readable Aliases & Descriptions

  • Attach memorable names like "Clean baseline before Foo v3 + new DB schema".
  • Aliases are stored in a local registry and embedded inside snapshots (as a marker file) and next to exported DMGs.
  • Persists even if you rename snapshots or move DMGs around.

3. Compressed DMG Export (Portable & Archival)

  • Export any snapshot (or just your Home folder) as a compressed UDZO .dmg.
  • Store on external drives, NAS, or cloud for long-term "golden images".
  • DMGs can be restored on the same or different Macs via Disk Utility or asr.

4. Safe, Guided Interactive Experience

The script provides a friendly menu with state inspection, automatic cleanup of stale mount points, and helpful warnings (e.g. when an external volume is mounted read-only).

Installation

git clone https://github.com/clawbunny/macos-clean-state.git
cd macos-clean-state
chmod +x clean-state.sh

# Optional: put it in your PATH
ln -s "$(pwd)/clean-state.sh" ~/bin/clean-state

Basic Usage

Just run it with no arguments for the interactive menu:

./clean-state.sh

Common CLI examples:

# Create a snapshot with a description
./clean-state.sh --create-snapshot --description "Clean baseline before running full test suite"

# Create a compressed DMG on an external drive
./clean-state.sh --create-dmg --dest /Volumes/MySSD --name "Clean-2026-06-05.dmg"

# Tag an existing snapshot
./clean-state.sh --tag-snapshot --description "Before installing new security tools"

# View current state and aliases
./clean-state.sh --status

Interactive Menu Options

9Refresh and reprint the current device state.
hShow this help text.
qQuit.

Destination Chooser (when creating a DMG)

  • Numbers (1, 2, ...): Real mounted volumes under /Volumes. Read-only volumes (common on external APFS "system" partitions) are clearly marked.
  • d: Your Desktop folder (always safe for testing).
  • o: Type any custom path (e.g. /Volumes/Sandisk\ -\ Data/Backups).

Restoring a Clean State

Fastest Method (Recommended)

  1. Restart and boot into Recovery (hold power button on Apple Silicon).
  2. Choose "Restore from Time Machine Backup".
  3. Select your internal drive and pick the snapshot by its timestamp or alias.
  4. Restore and reboot. You're back to the exact clean state.

From a DMG (Portable / Cross-Machine)

  1. Boot to Recovery.
  2. Open Disk Utility → View → Show All Devices.
  3. Use the Restore tab: source = your .dmg, destination = the internal APFS container.
  4. Or use the command line: asr restore --source /path/to/Clean.dmg --target /dev/diskX --erase
Important: Always run diskutil list in Recovery first to confirm disk identifiers. Restoring is destructive.

Tips & Best Practices

  • Create a snapshot after you have your environment exactly how you want it, but before you start testing.
  • Keep 1–2 recent local snapshots for speed + at least one compressed DMG on an external drive as your "gold master".
  • Use aliases liberally — they make it obvious which snapshot is which months later.
  • When using external drives, prefer the - Data volume over the main volume name (the latter is often the read-only system volume).
  • Grant Terminal "Full Disk Access" in System Settings → Privacy & Security if you encounter permission errors.

Requirements

  • macOS 10.13 High Sierra or later (APFS required)
  • Standard user with admin rights (for sudo operations)
  • Optional: external drive with enough free space for DMG archives
